# 知识内化完整方法论：从知识库到真正的改变

## 一、核心问题

知识库里有 ≠ 你懂了 ≠ 你会用了 ≠ 你变了。

本方法论解决三个断裂：

1. **知识库→脑子**：AI教你学会知识库里的内容
2. **脑子→行为**：把理解变成自动反应
3. **行为→输出**：把改变变成有价值的产出

## 二、知识类型识别——教学的前提

不同类型的知识需要完全不同的教法。在开始教学之前，必须先识别知识类型。

| 知识类型 | 特征 | 典型例子 | 核心教法 | 常见误区 |
|----------|------|----------|----------|----------|
| 概念性知识 | 抽象概念、理论框架 | 锚定效应、系统1/系统2 | 苏格拉底五步法 | 当定义背，不建构理解 |
| 程序性知识 | 有步骤、有流程 | 量学战法、写作SOP | 四步实操法 | 只看不做，以为看懂=会做 |
| 事实性知识 | 具体数据、事件 | 历史年份、统计数据 | 记忆术 | 反复阅读代替主动回忆 |
| 元认知知识 | 关于思维本身的知识 | 如何学习、如何决策 | 反思日记法 | 只学不用，不反思自己的思维 |

### 识别流程

```
从知识库中读取[知识卡片]，判断其知识类型：

1. 如果卡片包含"定义+机制+适用条件" → 概念性知识
2. 如果卡片包含"步骤+判断条件+操作要点" → 程序性知识
3. 如果卡片包含"数据+来源+时间" → 事实性知识
4. 如果卡片包含"关于如何思考/判断/决策" → 元认知知识

一张卡片可能同时属于多个类型，以主要类型决定教法。
```

### 混合类型的处理

```
很多知识是混合型的。处理规则：

1. 程序性+概念性 → 先教概念（为什么这样做），再教程序（怎么做）
2. 事实性+概念性 → 先记事实（用记忆术），再建构理解（用五步法）
3. 元认知+程序性 → 先反思（自己的思维模式），再给方法（改进步骤）

核心原则：理解先行，操作跟进，反思贯穿。
```

## 三、三阶段流程总览

```
知识库 → 阶段A：AI教学（AI当老师）→ 阶段B：内化（AI当陪练）→ 阶段C：输出（AI当助手）
           理解层次≥L3               内化度≥I3               产出物通过质量检查
```

## 四、阶段A：AI教学

### 4.1 课程自动生成

在开始教学之前，AI根据知识库自动生成课程计划。详见 [course-generation.md](references/course-generation.md)。

```
从知识库中读取所有知识卡片，生成课程计划：

第1步：识别所有概念的知识类型（概念性/程序性/事实性/元认知）
第2步：识别概念之间的依赖关系（A依赖B → 先学B）
第3步：识别概念之间的关联关系（对立组/递进组/互补组）
第4步：计算每个概念的"枢纽度"（被多少其他概念依赖）
第5步：生成学习顺序：
  - 第一批：枢纽度最高的3个概念
  - 后续批次：依赖已学概念的新概念
  - 关联概念安排在同一批次

输出格式：
| 批次 | 概念 | 类型 | 依赖 | 关联组 | 预计时长 |
|------|------|------|------|--------|----------|
| 1 | [概念A] | 概念性 | 无 | 对立组(A,B) | 45分钟 |
| 1 | [概念B] | 概念性 | 无 | 对立组(A,B) | 45分钟 |
| 2 | [概念C] | 程序性 | A | - | 60分钟 |
```

### 4.2 苏格拉底式五步教学法（概念性知识）

教一个概念的完整流程：激活→冲突→建构→检验→巩固。详见 [socratic-teaching.md](references/socratic-teaching.md)。

**第1步：激活——从人已知的东西开始**

人只能通过"已有知识"来理解"新知识"。不先了解人已知什么，教学要么太浅要么太深。

```
我要学习[概念]。在讲新概念之前，先了解我已知什么：

1. 描述一个与[概念]相关的生活体验（不提概念名称），问我有没有类似经历
2. 问我对这个现象的直觉理解
3. 发现我的"已知"和"误解"

关键：不提概念名称，只描述体验。让人先感受到现象，再命名。
```

AI做的事：
- 从知识库中提取概念的核心现象
- 把现象翻译成人能共鸣的生活场景
- 发现人的"已知"和"误解"

**第2步：冲突——制造认知失调**

没有冲突，人不会觉得需要学新东西。"我知道自己不理性"和"我亲眼看到自己有多不理性"是两回事。

```
设计一个让人"亲自踩坑"的体验：
1. 给出一个测试/问题，让人先给出判断
2. 揭示人的判断被什么影响了
3. 让人亲身体验"我以为我懂，其实我不懂"
4. 在人最困惑的时候，给出概念名称

关键：让人先犯错，再揭示。犯错比听讲更有教学价值。
```

**第3步：建构——用人的已知理解新概念**

直接给定义，人只能死记硬背。通过自己的体验建构理解，人才真正"拥有"这个知识。

```
现在我已经体验了[概念]，请引导我自己建构理解：
1. 问：刚才的体验中，是什么影响了你？（识别"锚"）
2. 问：它是怎么影响你的？（理解机制）
3. 问：你能想到生活中其他类似的例子吗？（迁移）
4. 让我用自己的话给[概念]下定义
5. 把我的定义和知识库中的定义对比，找出差异
6. 在差异处深挖

关键：不给定义，让人自己建构。差异就是理解不够深的地方。
```

**第4步：检验——你真的理解了吗？**

```
请用三类递进问题检验我对[概念]的理解：

问题1（辨析）：给出4个场景，其中2个涉及[概念]，2个涉及[相似概念]，
  让我判断每个场景属于哪个概念。

问题2（边界）：给出一个[概念]看似适用但实际不适用的场景，
  让我判断并解释为什么。

问题3（迁移）：让我描述一个自己工作中的真实场景，
  判断[概念]是否适用，如果适用，具体怎么用。

每个问题我先答，你再给反馈。错误比正确更有价值——错误暴露理解的盲区。
```

**第5步：巩固——确保不忘**

```
今天我学了[概念]，请帮我做三件事来巩固：

1. 一句话总结：让我用一句话说出[概念]是什么。
   如果我说不清楚 → 回到第3步重新建构。

2. 触发器设计：帮我把[概念]变成一个具体的触发器。
   不是"做决策时想想[概念]"（太模糊），
   而是"每当[具体情境]时，先问自己[具体问题]"。
   触发器比概念本身更实用。

3. 下次复习：告诉我下次复习的时间和方式。
   3天后你会问我："最近你有没有遇到[概念]适用的场景？"
   我不需要记住定义，只需要记住那个"踩坑的体验"。
   体验比定义更持久。
```

### 4.3 四步实操法（程序性知识）

程序性知识的核心是"做"，不是"懂"。看懂步骤 ≠ 会操作。详见 [procedural-teaching.md](references/procedural-teaching.md)。

```
第1步：示范——AI演示完整流程
  AI从知识库中提取程序，用具体案例走一遍完整流程。
  关键：每个判断点都要解释"为什么在这里判断，判断标准是什么"。

第2步：带做——AI给场景，人跟着做
  AI给出一个新场景，人按步骤操作，AI在每一步检查。
  关键：人必须亲手操作，不能只看AI做。
  人的操作和知识库中的步骤有偏差 → 偏差就是没理解的地方。

第3步：独立——人自己做，AI检查结果
  AI给出场景，人独立完成全部步骤，AI只检查最终结果。
  结果正确 → 进入第4步。
  结果错误 → 回到第2步，在出错步骤重新带做。

第4步：变式——AI给变异场景，人灵活应对
  AI给出和训练场景不同的变式（缺少信息/条件变化/多步骤组合）。
  人需要自己判断怎么调整步骤。
  这是程序性知识从L3到L4的关键——不是死记步骤，而是理解步骤背后的逻辑。
```

### 4.4 记忆术（事实性知识）

事实性知识的核心挑战是遗忘。反复阅读效率极低，必须用主动回忆。详见 [factual-memorization.md](references/factual-memorization.md)。

```
方法1：联想锚定
  把新事实和已有记忆绑定。
  "1929大萧条" → 联想"29=二舅，二舅在大萧条中破产"

方法2：空间记忆法（记忆宫殿）
  把事实放在熟悉的空间位置上。
  适合需要记忆大量有序事实的场景。

方法3：间隔主动回忆
  不看笔记，主动回忆 → 比反复阅读效率高300%。
  间隔：当天 → 1天 → 3天 → 7天 → 14天 → 30天
  每次回忆后：能回忆 → 间隔翻倍；回忆失败 → 间隔重置

方法4：测试效应
  每次复习不是"再看一遍"，而是"先测试，再看答案"。
  测试本身比阅读更能巩固记忆。
```

### 4.5 反思日记法（元认知知识）

元认知知识的核心不是"知道"，而是"觉察自己的思维"。详见 [metacognitive-reflection.md](references/metacognitive-reflection.md)。

```
每周反思模板：

1. 本周我做了一个什么决策？
2. 我当时是怎么思考的？（还原思考过程）
3. 知识库中的[元认知概念]会怎么分析我的思考过程？
4. 我的思考过程中有哪些偏差/盲点？
5. 下次遇到类似情况，我会怎么调整？

AI的角色：不是教你怎么思考，而是像镜子一样照出你的思维模式。
关键：必须用自己真实的决策，不能用假设场景。
```

### 4.6 六种教学武器

详见 [teaching-weapons.md](references/teaching-weapons.md)。

| 武器 | 解决层次 | 适用知识类型 | 核心方法 |
|------|----------|-------------|----------|
| 对比辨析 | L1识别 | 概念性 | 给场景判断属于A还是B |
| 费曼检验 | L2复述 | 概念性/程序性 | 用12岁孩子能懂的话解释 |
| 场景推演 | L3应用 | 概念性/程序性 | AI给场景，人用概念分析/操作 |
| 逆向激活 | L4迁移 | 所有类型 | AI给真实场景，人判断适用性 |
| 组合挑战 | L5创造 | 概念性/元认知 | 两个概念组合解释新现象 |
| 魔鬼考试 | 综合检验 | 所有类型 | 5题综合测试，20分以上通过 |

### 4.7 AI实时调整——五种情况

教学不是按剧本走，必须根据人的反应实时调整。详见 [realtime-adjustment.md](references/realtime-adjustment.md)。

```
情况1：人回答得太快太流畅
  → 可能只是表面记忆，不是真正理解
  → 调整：换一个角度问同一个概念，或给一个陷阱场景

情况2：人反复在同一个点犯错
  → 不是"再讲一遍"能解决的，需要换一种建构方式
  → 调整：换一个类比/换一个体验场景/回到冲突步骤重新制造认知失调

情况3：人表现出明显的不耐烦
  → 可能认知负荷过高，或者这个概念对人不重要
  → 调整：拆成更小的概念/先跳过，标记为"稍后回来"/问人是否需要暂停

情况4：人提出了知识库中没有的见解
  → 可能是真正理解后的创新，也可能是误解
  → 调整：先验证（用三重验证法），如果是创新 → 更新知识库

情况5：人把多个概念混淆了
  → 说明概念之间的边界不够清晰
  → 调整：暂停当前概念，用对比辨析武器把混淆的概念一起重新学
```

### 4.8 理解层次

| 层次 | 含义 | 检验方式 |
|------|------|----------|
| L1 识别 | 看到概念能认出来 | 从多个场景中选出涉及该概念的 |
| L2 复述 | 能用自己的话说出来 | 不看笔记，用自己的话解释 |
| L3 应用 | 能在给定场景中使用 | 给场景，用概念分析 |
| L4 迁移 | 能在新场景中自发调用 | 描述自己的真实场景，判断概念是否适用 |
| L5 创造 | 能基于此产生新见解 | 组合两个概念，解释新现象 |

**阶段A出口条件**：核心概念理解层次≥L3，可进入阶段B。

### 4.9 概念分组策略

| 分组类型 | 说明 | 示例 |
|----------|------|------|
| 对立组 | 互为对立面，一起学对比更清晰 | 锚定效应 vs 理性决策 |
| 递进组 | 前者是后者的基础，按顺序学 | 系统1 → 系统2 |
| 互补组 | 组合应用更有力，一起学 | 锚定效应 + 框架效应 |
| 独立组 | 不依赖其他概念，随时可学 | 幸存者偏差 |

### 4.10 诊断先行

在开始教学之前，先诊断人当前的理解程度。

```
从知识库中选取[概念]，请按以下流程诊断我的理解程度：

第1题（L1识别）：从4个场景中选出哪个涉及[概念]
第2题（L2复述）：用自己的话解释[概念]，不能看笔记
第3题（L3应用）：给定一个场景，用[概念]分析
第4题（L4迁移）：描述一个我最近遇到的场景，判断[概念]是否适用
第5题（L5创造）：[概念]和[另一个概念]组合，能解释什么新现象？

根据我的回答，判断我当前的理解层次，并给出下一步学习建议。
```

分层学习策略：

| 理解层次 | 缺失表现 | AI教学策略 |
|----------|----------|-----------|
| L1 识别 | 混淆相似概念 | 对比辨析 |
| L2 复述 | 只能背定义 | 费曼检验 |
| L3 应用 | 给定场景不会用 | 场景推演 |
| L4 迁移 | 只会在书中场景用 | 逆向激活 |
| L5 创造 | 无法组合产生新见解 | 组合挑战 |

## 五、阶段B：内化

### 5.1 内化闭环五步

详见 [internalization-loop.md](references/internalization-loop.md)。

```
知识库 → 场景激活 → 尝试应用 → 失败/成功 → 反思归因 → 修正理解 → 知识库
```

理解（L3以上）是内化的前提。但理解和内化之间有一个关键断裂：你能在给定场景中用概念（L3），不代表你会在真实生活中主动想起它（I3）。

内化要解决的核心问题：**从"被提示时能用"到"场景出现时自动调用"。**

### 5.2 场景激活

知识在知识库里是"死的"，只有被场景激活才是"活的"。

```
从我的知识库中，找出[概念]，并为它生成3个我可能遇到的真实场景。
要求：
1. 场景必须具体到我明天就可能遇到
2. 难度递进：
   场景1（显而易见的应用）→ 场景2（需要判断才知该用）→ 场景3（表面看不该用但其实该用）
3. 每个场景给出：
   - 情境描述
   - 我会本能做出的反应
   - 用[概念]后应该做的反应
```

**触发器设计**

从抽象概念到具体触发器：

```
错误触发器："做决策时想想[概念]"（太模糊，不会触发）
正确触发器："每当[具体情境]时，先问自己[具体问题]"

示例：
  概念：锚定效应
  正确触发器："每当看到任何数字时，先问'这个数字从哪来的'"

  概念：确认偏误
  正确触发器："每当我觉得'果然如此'时，先找一条反面证据"

  概念：沉没成本
  正确触发器："每当想说'已经投入这么多了'时，问自己'如果从零开始，我还会选这个吗'"
```

### 5.3 尝试应用

**两种应用模式**

模式A：真实场景应用（优先）

```
当你在生活中遇到与[概念]相关的场景时：
1. 先记录你的本能反应
2. 再用[概念]重新判断
3. 记录两个判断的差异
4. 选择按哪个判断行动
5. 记录结果
```

模式B：AI模拟场景（补充）

```
假设我正在[场景]，[具体情况]发生了，我会怎么做？

我回答后：
- 指出我的回答中哪些部分受到了[偏差/惯性]的影响
- 如果用[概念]重新审视，我会怎么调整
- 让我给出调整后的回答
- 评估这次是否更好，是否还有盲点
```

**应用记录模板**

```
日期：[...]
场景：[...]
概念：[...]
本能反应：[我想做X]
概念指导：[应该做Y]
实际行动：[我做了X/Y/其他]
结果：[...]

结果分类：
  □ 成功：按概念指导行动，结果符合预期
  □ 部分成功：按概念指导行动，但结果不如预期
  □ 失败-未触发：遇到场景但没想起概念
  □ 失败-触发了没用：想起了概念但没按它行动
  □ 失败-用了但无效：按概念行动了但结果不好

最常见的是"失败-未触发"——你存了卡片，但场景出现时根本没想起来。
```

### 5.4 反思归因

```
AI辅助反思指令：

我遇到了以下情况：[描述应用记录]
请帮我分析：

1. 如果是"未触发"：
   → 这个概念的触发条件是否太抽象？
   → 能不能设计一个更具体的触发器？

2. 如果是"触发了没用"：
   → 是惯性太强？还是概念不够清晰？
   → 能不能设计一个"最小行动"？
   → 不是"完全按概念行动"，而是"先暂停3秒，再决定"

3. 如果是"用了但无效"：
   → 概念的适用边界是否判断错了？
   → 需要回到阶段A重新学习这个概念吗？

4. 如果是"成功"：
   → 成功是因为概念有效，还是巧合？
   → 能不能提炼出可复制的模式？
```

### 5.5 修正理解

```
修正类型：

□ 修正触发条件：
  旧触发："做决策时想想[概念]"
  新触发："当[具体情境]时，先问[具体问题]"
  → 更新知识卡片的"行动提示"字段

□ 修正适用边界：
  旧边界："适用于所有涉及X的场景"
  新边界："适用于A场景，不适用于B场景"
  → 更新知识卡片的"边界"字段

□ 修正理解深度：
  旧理解："X就是Y"
  新理解："X不只是Y，还包括Z"
  → 更新知识卡片的"定义"和"论证链"

□ 降级处理：
  如果一个概念反复应用失败：
  → 降低confidence值
  → 标记为"待重新理解"
  → 回到阶段A重新学习
```

### 5.6 双重验证机制

AI的评估有局限——AI无法真正知道你"是否内化了"，只能通过你的语言表现来推断。因此需要双重验证。详见 [dual-validation.md](references/dual-validation.md)。

```
第一重：AI评估（语言层面）
  AI通过你的回答判断理解层次和内化度。
  局限：人可以说出正确的答案但不真正理解（应试型回答）。
  适用：阶段A的理解层次评估。

第二重：行为验证（行动层面）
  通过你在真实场景中的行为来验证内化度。
  方法：
    1. 场景出现时你是否主动想起了概念？（I2→I3的关键）
    2. 想起后你是否按概念行动了？（I3→I4的关键）
    3. 行动后结果是否符合预期？（验证理解是否正确）
  适用：阶段B的内化度评估。

双重验证规则：
  - AI评估L3 + 行为验证I3 = 真正达到出口条件
  - AI评估L3 + 行为验证I1 = 理解了但没内化，不能进入阶段C
  - AI评估L5 + 行为验证I0 = 高分低能，需要回到阶段B

行为验证的记录方式：
  每周回顾应用记录，统计：
  - 主动想起次数（不是AI提示后想起）
  - 按概念行动次数
  - 行动后结果符合预期的比例
  这三个数据比AI的评估更可靠。
```

### 5.7 知识衰减监控

知识不用就会衰减。衰减不是线性的，而是分阶段的。详见 [decay-monitoring.md](references/decay-monitoring.md)。

```
衰减三阶段：

阶段1：模糊期（7-14天不用）
  表现：能想起概念名称和大意，但细节模糊
  应对：一次快速复习即可恢复（5-10分钟）
  AI动作：在复习日提醒，给一个场景快速检验

阶段2：失活期（14-30天不用）
  表现：需要提示才能想起，无法主动调用
  应对：需要重新做一次场景推演（15分钟）
  AI动作：生成场景推演练习，检查是否能应用

阶段3：遗忘期（30天以上不用）
  表现：完全想不起来，和没学过一样
  应对：需要重新用五步法学习（但比第一次快，因为有残留记忆）
  AI动作：标记为"需重新学习"，安排五步法教学

衰减监控规则：
  - 每个概念记录"最后使用日期"
  - 超过7天未使用 → 标记为"模糊期"
  - 超过14天未使用 → 标记为"失活期"
  - 超过30天未使用 → 标记为"遗忘期"
  - 每周生成"衰减报告"，列出需要复习的概念
```

### 5.8 内化加速技术

**间隔召回——对抗遗忘曲线**

```
AI驱动的间隔召回计划：

第1次：学完当天
  AI问：用一句话说出[概念]是什么？

第2次：3天后
  AI问：[概念]在你最近遇到的哪个场景中可能适用？

第3次：1周后
  AI给一个场景，问：在这个场景中，[概念]会建议你怎么做？

第4次：2周后
  AI问：[概念]和[另一个概念]有什么区别？什么时候用哪个？

第5次：1个月后
  AI问：你上次应用[概念]是什么时候？结果如何？你会怎么改进？

判断标准：
  每次都能流畅回答 → 概念已进入长期记忆
  某次卡壳 → 回到知识卡片重新理解，重置间隔
```

**对比练习——区分相似概念**

```
AI生成的对比练习：

概念A：[锚定效应]
概念B：[确认偏误]

场景1：你看到一个产品的好评，然后更倾向于买它 → 确认偏误
场景2：你看到一个产品的定价999，然后觉得499很便宜 → 锚定效应
场景3：你买了一支股票后，只关注利好消息 → 确认偏误
场景4：你问朋友"你觉得这个值多少钱"后，自己的估价被影响了 → 锚定效应

练习方式：AI给出场景，人判断是哪个概念，AI给出反馈。
```

**极端场景测试——检验内化深度**

```
AI生成的极端场景：

场景1：[概念]和你的直觉完全相反的情况 → 你还会用吗？
场景2：[概念]和另一个你同样认同的概念冲突 → 你怎么选？
场景3：用[概念]行动可能让你在社交中显得奇怪 → 你还敢用吗？
场景4：[概念]告诉你不该做一件你非常想做的事 → 你能忍住吗？

这些极端场景才是内化的真正测试。
如果只在简单场景中能用，那只是"知道"，不是"内化"。
```

### 5.9 内化度评估

```
内化度分级：

I0 外部存储：知识在知识库里，不在脑子里
I1 可回忆：被提示时能说出，但不会主动想起
I2 可触发：遇到场景会想起，但不一定按它行动
I3 可应用：遇到场景能按概念行动，但需刻意努力
I4 已内化：变成自动反应，不需要刻意想起
I5 已融合：重塑了身份认知

自测方法（不看任何笔记）：
  1. 最近一周，你在什么场景中想起了这个概念？→ 想不起 = I1
  2. 想起后，你按它行动了吗？→ 没行动 = I2
  3. 行动后，结果如何？→ 结果不好 = I3
  4. 下次遇到类似场景，你还需要刻意想起它吗？→ 需要 = I4
  5. 如果有人问你是谁，这个概念会影响你的回答吗？→ 不会 = I4，会 = I5

目标：核心概念至少达到 I3，改变生活的概念至少达到 I4。
```

**阶段B出口条件**：核心概念内化度≥I3（双重验证通过），可进入阶段C。

## 六、阶段C：输出

### 6.1 输出前提

**内化是输出的前提。** 跳过内化直接输出，产出的内容是"搬运"而不是"洞察"。详见 [output-strategies.md](references/output-strategies.md)。

| 维度 | 未内化的输出 | 内化后的输出 |
|------|-------------|-------------|
| 观点 | "作者认为X" | "我实践了X，发现它在Y条件下有效" |
| 举例 | 引用书中的案例 | 用自己的真实经历 |
| 判断 | "这本书很好" | "这个观点我同意/不同意，因为我试过了" |
| 深度 | 能复述论证链 | 能指出论证链在现实中的断裂点 |
| 独特性 | 任何AI都能生成 | 只有经历过的人才能写出 |

**入口条件**：核心概念内化度≥I3。

### 6.2 四类输出

| 输出类型 | 目的 | 人的核心 | AI的核心 | AI参与度 |
|----------|------|----------|----------|----------|
| 行动型 | 改变自己 | 定方向 | 设计执行 | 40% |
| 表达型 | 影响他人 | 出观点 | 做结构 | 40% |
| 创造型 | 产生新知 | 做组合 | 做扩展 | 60% |
| 教学型 | 教会他人 | 做示范 | 做路径 | 50% |

### 6.3 行动型输出

协作流程：

```
第1步（人）：从迁移规则中选出最想落地的那一条
第2步（AI）：生成行动方案（行动清单+失败预判+验证方式）
第3步（人）：审查并承诺（修改方案，加入自己的约束）
第4步（AI+人）：迭代（1周后人反馈执行情况，AI修正方案）
```

行动方案模板：

```
【行动方案 AP-XXX】

来源：[书名] → [概念] → [迁移规则MR-XXX]

行动清单：
  □ 当[触发场景]时，执行[检查项1]
  □ 在[决策前]，先问自己[问题]
  □ 每周[时间]回顾[指标]

失败预判：
  最可能忽略的环节：[...]
  最可能放弃的原因：[...]
  对策：[...]

验证方式：
  [时间]后检查[指标]，预期[结果]

承诺：我从[日期]开始执行
```

### 6.4 表达型输出

协作流程：

```
第1步（人）：确定选题和核心观点
第2步（AI）：从知识卡片中提取素材，生成大纲
第3步（人）：修改大纲，加入个人声音
第4步（AI）：根据修改后的大纲生成初稿
第5步（人）：重写关键段落（开头、核心论证、结尾）
第6步（AI+人）：交替打磨
```

关键铁律：

```
AI写的文章是"任何人都能写的"，
你写的文章是"只有你能写的"。

判断标准：去掉你的名字，这篇文章还有任何理由让人知道是你写的吗？
如果没有 → 你只是在搬运，不是在表达。
```

### 6.5 创造型输出

协作流程：

```
第1步（人）：发现跨领域/跨书的连接点
第2步（AI）：验证连接的逻辑性，搜索支撑证据或反例
第3步（人）：提出新框架
第4步（AI）：完善框架（完整定义+适用场景+边界+可测试预测）
第5步（人+AI）：迭代验证（人用真实场景测试，AI搜索反例）
```

创新判断标准：

```
你的新框架/方法是否满足：
  □ 能解释旧框架解释不了的现象？
  □ 能预测旧框架预测不了的结果？
  □ 能指导旧框架指导不了的行动？

如果3个都是"否" → 不是创新，只是换了个说法。
如果有1个"是" → 值得继续发展。
```

### 6.6 教学型输出

协作流程：

```
第1步（人）：确定教学目标
第2步（AI）：从知识卡片生成学习路径
第3步（人）：加入示范案例（用自己的真实经历）
第4步（AI）：生成教学材料
第5步（人+AI）：模拟教学（AI扮演学习者提问，人回答）
```

教学核心原则：

```
不要只展示结论，要展示思考过程。
不要只讲成功案例，要讲踩坑经历。
不要只给正确答案，要让人自己犯错。

最好的教学不是"我来告诉你"，
而是"我踩过这个坑，让我带你绕过去"。
```

## 七、六条教学铁律

1. **永远不要先给定义，先给体验**——让人先踩坑，再命名体验
2. **永远不要只讲正确答案，先让人犯错**——错误比正确更有教学价值
3. **永远不要一次讲太多，一个概念一个概念来**——认知负荷有限
4. **永远用人的话，不用书中的话**——"你被第一个数字带偏了"优于"过度依赖最先获得的信息"
5. **每次教学必须包含"你自己的场景"**——抽象理解≠能用到自己身上
6. **教学结束必须设触发器**——"每当看到数字，先问从哪来的"优于"记住锚定效应"

## 八、动机管理三原则

学习最大的敌人不是难度，而是放弃。动机管理贯穿三个阶段。

### 原则一：最小启动

```
降低开始学习的门槛：

1. 每次学习开始前，AI给出"今天只需做一件事"：
   不是"学习锚定效应"（太大），
   而是"回答一个问题：你上次买东西时，有没有被标价影响？"（只需1分钟）

2. 永远不要一次安排超过30分钟的学习任务

3. 如果人表现出犹豫，先做最简单的诊断题（1分钟），
   让人发现自己"其实没那么懂"，产生学习动力
```

### 原则二：即时反馈

```
让每次学习都有明确的收获感：

1. 每个概念学完后，AI给出理解层次判断：
   "你现在对[概念]的理解达到了L3（应用层），比开始时的L1提升了2个层次"

2. 每次内化练习后，AI给出内化度判断：
   "你在真实场景中主动想起了[概念]，内化度从I1提升到I2"

3. 每次输出后，AI给出质量评估：
   "这篇文章的个人痕迹评分：7/10，比上次提升了2分"

关键：反馈必须具体，不能只是"很好""不错"。
```

### 原则三：进度可见

```
让人看到自己的整体进步：

1. 学习进度看板：
   | 概念 | 教学状态 | 理解层次 | 内化度 | 输出状态 |
   |------|----------|----------|--------|----------|
   | [A] | ✅ 完成 | L4 | I3 | 行动型✅ |
   | [B] | ✅ 完成 | L3 | I2 | 进行中 |
   | [C] | 🔜 待学 | - | - | - |

2. 整体完成度：
   已完成概念数 / 总概念数 = XX%
   已内化概念数 / 已学概念数 = XX%

3. 里程碑：
   "你已经学会了5个概念，超过了80%的读者"
   "你已经在真实场景中应用了3次，超过了90%的读者"
```

## 九、会话恢复协议

学习是跨多次会话的，必须解决上下文断裂问题。详见 [session-recovery.md](references/session-recovery.md)。

### 9.1 会话保存

```
每次学习会话结束时，AI自动保存：

1. 学习快照：
   - 当前学习的概念及进度
   - 理解层次和内化度
   - 待解决的问题
   - 下一步计划

2. 保存位置：知识库目录下的 session-snapshot.md

3. 保存格式：
   ## 会话快照 YYYY-MM-DD HH:MM

   当前概念：[概念名]
   教学步骤：[第X步/共5步]
   理解层次：L[X]
   内化度：I[X]
   待解决：[具体问题]
   下次计划：[具体步骤]

   ### 本次关键对话
   - 人的核心误解：[...]
   - 人的建构定义：[...]
   - 与知识库定义的差异：[...]
```

### 9.2 会话恢复

```
每次新会话开始时，AI自动执行：

1. 读取 session-snapshot.md
2. 向人确认：
   "上次我们学到[概念]的[第X步]，你的理解达到了L[X]。
    你当时对[具体问题]还有疑问。
    我们从哪里继续？"
3. 给出选项：
   a. 从上次断点继续
   b. 先快速回顾再继续
   c. 跳到新概念

4. 如果超过7天未学习：
   - 先做衰减评估（见5.7知识衰减监控）
   - 根据衰减程度决定是从头学还是快速复习
```

## 十、完成度评估——四级标准

不是所有概念都需要学到L5/I5。根据概念的重要性，设定不同的完成度目标。

```
四级完成度：

铜级（基础认知）：
  理解层次 ≥ L2（能复述）
  内化度 ≥ I1（可回忆）
  适用：辅助性概念、背景知识
  标准：能说出概念是什么，能在提示下回忆

银级（功能掌握）：
  理解层次 ≥ L3（能应用）
  内化度 ≥ I3（可应用）
  适用：核心概念、常用工具
  标准：能在给定场景中正确使用，能在真实场景中主动想起并行动
  → 这是大多数概念的完成目标

金级（深度内化）：
  理解层次 ≥ L4（能迁移）
  内化度 ≥ I4（已内化）
  适用：改变思维模式的核心概念
  标准：能在新场景中自发调用，变成自动反应

钻石级（融会贯通）：
  理解层次 ≥ L5（能创造）
  内化度 ≥ I5（已融合）
  适用：重塑世界观的根本概念
  标准：能组合创新，已融入身份认知
```

### 完成度分配规则

```
AI根据概念的重要性自动建议完成度目标：

1. 枢纽概念（被最多概念依赖）→ 金级
2. 核心概念（直接改变行为）→ 银级
3. 辅助概念（帮助理解核心）→ 铜级
4. 世界观概念（重塑认知框架）→ 钻石级

人可以调整每个概念的完成度目标。
未达到目标的概念 → 继续学习。
超过目标的投入 → 提醒人"这个概念已经够用了，时间花在更重要的概念上"。
```

## 十一、学习节奏

### 11.1 单概念学习

```
第1天：五步法教学（30-60分钟）
第2天：对比辨析 + 费曼检验（15分钟）
第4天：场景推演（15分钟）
第7天：逆向激活（10分钟）
第14天：组合挑战（15分钟）
第30天：魔鬼考试（20分钟）
```

### 11.2 多概念并行

```
每次学习不超过3个概念
优先学习"枢纽概念"（被最多其他概念依赖的）
有关联的概念一起学（对立组/递进组/互补组）
每周做一次跨概念组合挑战
```

### 11.3 间隔复习

```
每次复习后：
  3题都能流畅回答 → 下次间隔翻倍
  1题卡壳 → 间隔重置为3天
  2题以上卡壳 → 需要重新用五步法学习
```

## 十二、反馈机制

```
阶段A中发现理解错误 → 回到book-extraction的阶段二重新追问
阶段B中应用失败 → 回到阶段A重新学习该概念
阶段C中发现新问题 → 回到book-extraction的阶段二重新追问
阶段B中的修正 → 更新知识库中的卡片
```

## 十三、质量检查

### 理解检查（阶段A出口）

- [ ] 能用自己的话说出每个核心概念的定义
- [ ] 能区分相似概念的关键差异
- [ ] 能在给定场景中正确应用概念
- [ ] 能指出概念的适用边界

### 内化检查（阶段B出口）

- [ ] 最近一周在真实场景中想起过概念
- [ ] 想起后按概念行动了
- [ ] 行动后结果符合预期
- [ ] 不需要刻意提醒就能想起
- [ ] 行为验证数据支持（不是只有AI评估）

### 输出检查（阶段C出口）

- [ ] 行动型：行动项具体到明天就能做，有触发条件，有验证方式
- [ ] 表达型：去掉你的名字，文章还有个人痕迹吗
- [ ] 创造型：新框架能解释旧框架解释不了的现象吗
- [ ] 教学型：学习者看完后能独立操作吗

## 十四、非书籍知识源适配

本方法论虽然从读书出发，但同样适用于其他知识源。

```
适配规则：

论文/学术文章：
  → 阶段A重点：论证链拆解（前提→推理→结论→局限）
  → 额外步骤：识别学术争论点，对比不同学派观点

视频课程：
  → 阶段A重点：提取操作步骤和判断条件
  → 额外步骤：暂停→预测→对比讲师的下一步

实践经验：
  → 跳过阶段A的部分步骤（已有体验）
  → 阶段B重点：从经验中提炼隐性知识，显性化为卡片
  → 额外步骤：反思"我为什么这样做"，把直觉变成可迁移的规则

对话/访谈：
  → 阶段A重点：提取对方的思维模型和决策逻辑
  → 额外步骤：对比自己和对方的思维差异

核心原则：不管知识从哪来，三阶段流程不变——教学→内化→输出。
变的只是阶段A的"激活"步骤：书籍用书中案例激活，经验用自己的经历激活。
```
